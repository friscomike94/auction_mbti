# 경매 검색 기준 매핑

진단 결과(경매 유형 + 입력값)를 **법원경매에서 바로 거를 수 있는 검색 필터**로 변환한다. 핵심은 결정적 함수: `검색조건 = f(유형 축, profile 입력값)`.

## 1. 두 종류의 입력 구분

- **입력값에서 직접 오는 조건**(사용자가 고른 사실): 지역(투자지역), 가용자금, 규제지역 여부, 보유상황, 용도. → 검색조건의 뼈대.
- **유형 축에서 오는 취향**(성향): 자금성향(s/b), 난이도(c/x), 지역전략(n/w), 용도지향(r/f). → 물건종류·유찰횟수·권리난이도 등 "어떤 물건을 좋아하나"를 결정.

예) "경기·비규제·5천만원" = 입력값. "아파트·유찰1회·권리단순" = 유형(A-scnr: 소액·안전·연고지·실수요)에서 도출.

## 2. 검색 차원 (표준 출력 필드)

법원경매(courtauction.go.kr)에서 실제 거를 수 있는 필드 위주로 산출한다:

| 차원 | 도출 근거 |
|---|---|
| `region` (시/도 + 시군구) | 투자지역 입력 + 지역전략 축(연고지 n=특정 시군구 / 전국 w=광역 or 복수) |
| `regulated_preference` (비규제선호/무관/규제도가능) | 보유상황 + 규제여부 + 00_regulations.md(다주택→비규제선호, 규제지역 경매는 토허면제 기회) |
| `property_type` (아파트/오피스텔/다세대/빌라 등) | 용도지향 + 자금. 실수요 r→아파트 중심, 단타 f→환금성(아파트·오피스텔), 소액 s→다세대·빌라까지 |
| `price_max` / `price_min` (최저가 기준) | 가용자금 → 레버리지·규제 대출한도 반영(§3) |
| `min_fail_count` / `max_min_price_ratio` (유찰횟수/최저가율) | 자금규모 + 난이도. 소액·안전→유찰 1~2회로 내려온 물건, 특수물건→유찰 많아도 OK |
| `rights_difficulty` (안전/특수허용) | 난이도 축 c→말소기준 명확·임차인 단순·인수권리 없음 / x→유치권·법정지상권·선순위임차인 등 허용 |
| `area_pyeong` (면적/평형) | 용도 + 자금. 실수요→실거주 평형, 단타→환금 잘 되는 국민평형 |
| `notes` (주의 메모) | 세금·실거주·명도·자금조달 — 00_regulations.md·strategy_guide 연계 |

사이트에서 직접 못 거르는 지표(예: 예상 명도난이도)는 `notes`로 분리한다.

## 3. 자금 → 가격대 환산 (레버리지·규제 반영)

가용자금만으로 가격대를 정하지 말고:

1. **자금성향**: 소액분산 `s` → 자기자본 대비 보수적 상한(레버리지 적게), 다건 가능하게 가격대 낮게. 대액집중 `b` → 레버리지 적극, 상한 높게 소수 물건.
2. **규제 대출한도**(00_regulations.md): 규제지역 무주택 LTV 40%, 다주택·1주택 추가구입 주담대 금지(0%) 가능 → 그 경우 **자기자본만으로** 입찰가 상한 산출(대출 0 가정). 비규제 LTV 70%면 자금×레버리지로 상한 확대.
3. 경락잔금대출도 대출규제 동일 적용(D절) → 규제지역 경매는 대출 막혀 자기자본 부담↑를 가격대에 반영하고 `notes`에 경고.

예) 5천만원 + 소액분산 + 비규제 → 자기자본 5천 + 경락대출 약 70% 가정 시 최저가 1.5억~2억 내외 물건까지(보수적 제시 + "대출·DSR 확인" 메모). 같은 5천이 규제지역+다주택이면 대출 0 가정 → 5천 내외 물건 + 강한 경고.

## 4. 규제 연계 규칙

- **다주택 + 비규제 선호**: 취득세·양도세 중과 회피 관점에서 비규제 지역·저가 물건 가중. `regulated_preference=비규제선호`, notes에 취득세 중과 경고.
- **규제지역 경매의 양면**: 토허 실거주의무·갭투자차단·자금조달계획서 면제 → 규제지역인데 실거주의무 없이 살 수 있는 기회. 단 수요 쏠려 고가낙찰·대출규제는 동일. notes에 양면 명시(00_regulations.md D절).
- **무주택 실수요**: 규제지역도 생애최초·실거주 목적이면 적합. LTV 40%·전입의무 notes.
- 규제 수치·지역은 **반드시 `_workspace/00_regulations.md`만 근거**. 불확실 항목은 "확인 필요"로. 면책 유지.

## 5. 출력

- `_workspace/04_search_criteria.md` — 매핑 규칙(차원별 도출 로직), 핵심 유형별 검색조건 예시, 규제 연계.
- `_workspace/04_search_criteria.json` — 기계가 읽는 매핑 규칙 + 골든 케이스. 구조 예:

```json
{
  "version": "1.0",
  "dimensions": ["region","regulated_preference","property_type","price_min","price_max","min_fail_count","rights_difficulty","area_pyeong","notes"],
  "rules": {
    "price": { "by_fund_band": {...}, "leverage_by": "fund_axis & region_regulated & ownership" },
    "property_type": { "by_purpose_axis": {...}, "by_fund": {...} },
    "rights_difficulty": { "c": "safe", "x": "special_ok" },
    "min_fail_count": { "by_fund_and_difficulty": {...} },
    "region": { "from_input": "region", "by_region_axis": {...} },
    "regulated_preference": { "by_ownership_and_region": {...} }
  },
  "golden": [
    { "type": "A-scnr", "inputs": {"fund":"~5천","region":"경기","region_regulated":"normal","ownership":"none"},
      "expect": {"region":"경기(연고 시군구)","property_type":["아파트"],"price_max":"...","min_fail_count":">=1","rights_difficulty":"safe","notes":["..."]} }
  ]
}
```

웹은 이 규칙을 그대로 JS로 재현해 결과화면 카드 + JSON export를 만든다. 골든 케이스로 일치 검증한다.

## 6. 검증

- 모든 검색조건이 유형 축·입력값과 모순 없는가(특수물건 선호인데 권리단순만? 소액인데 고가대?).
- 모든 필드가 법원경매 사이트에서 거를 수 있는가(아니면 notes로).
- 규제 수치가 00_regulations.md에만 근거하는가. 면책 유지.
