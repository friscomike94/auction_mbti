# 인터랙티브 웹 테스트 빌드

진단 JSON 데이터 계약을 받아 사용자가 직접 푸는 단일 HTML 테스트를 만든다. 서버 불필요(본인 판단용), self-contained.

## 1. 데이터 주도 원칙

문항·선택지·유형을 코드에 하드코딩하지 않는다. `02_maemae.json`, `02_gyeongmae.json`을 읽어 동적 렌더링한다. 데이터가 바뀌어도 렌더/채점 로직은 그대로여야 한다. 단일 파일을 원하면 JSON을 `<script>` 안에 인라인 객체로 임베드한다(파일 경로 의존 제거).

JSON 구조는 `diagnostic-engine/references/data-schema.md` 참조. axes/questions/profile_inputs/types 구조를 정확히 가정한다.

## 2. 채점 = scoring.md 충실 재현

`02_scoring.md`의 규칙을 그대로 JS로 옮긴다. 핵심:
- 축별 양극 점수 합산 → 우세 극 채택
- 동점 시 `axes[].default_pole` 채택 + "경계형" 표시
- `axis_order` 순서로 극 코드 조합 + `code_prefix` → 유형 코드
- 경매: profile_inputs 조합으로 경고 플래그 산출

구현 후 scoring.md의 **골든 케이스(샘플 응답→기대 유형)**를 콘솔에서 돌려 일치를 확인한다. 불일치면 scoring.md가 정본이다.

## 3. 화면 구조

```
[시작] 트랙 선택 (매매 / 경매)
  → 경매 선택 시 profile_inputs(자금·용도·지역·보유·경험) 먼저 또는 문항에 섞어
[진행] 문항 1개씩 또는 페이지 단위, 진행률 표시, 이전/다음
[결과]
  - 유형 코드 + 유형명 + 키워드 칩 + summary
  - 강점 / 약점 / 맹점(⚠️) — 03_profiles.md에서
  - 매매 또는 경매 활용 가이드: 적합물건·함정·체크리스트 — 03_strategy_guide.md에서
  - 경매: 개인화 경고 플래그(다주택 세금, 초보 특수물건 등)
  - [다시하기] [결과 저장/공유]
```

상세 콘텐츠(강점/약점/가이드)는 `03_*.md`에서 가져와 유형 코드별로 인라인 데이터에 합치거나, 마크다운 헤더(`## [유형코드]`)로 파싱해 임베드한다.

## 3-1. 규제 입력 & 규제 경고 표시

진단 데이터에 `region_regulated` 등 규제 profile 입력이 있으면 시작/입력 단계에서 사용자가 고르게 한다(규제지역/비규제/모름). 결과 화면에는 규제 경고 플래그(다주택 대출금지·세금중과·토허 의무·경매 면제 등)를 level별(danger/warn/info) 색·아이콘으로 표시하고, `03_strategy_guide.md`의 규제 요약 섹션을 결과에 함께 노출한다. `[불확실]`/"확인 필요" 문구와 면책(전문가 확인)을 그대로 보존한다 — 규제 수치를 임의로 단정하지 않는다.

## 4. 결과 재현/공유

- 결과를 URL 쿼리(`?type=M-SHLE`) 또는 localStorage로 저장해 다시 볼 수 있게 한다.
- 본인용이므로 SNS 공유 버튼은 선택. 캡처/인쇄 친화 레이아웃이면 충분.

## 5. 디자인 톤

`algo-design` 스킬을 먼저 확인한다(크림 배경, Poppins+Lora, 오렌지/블루/그린 액센트, 카드·세그먼트 칩 컴포넌트, 잔잔한 모션). 사용자가 다른 톤을 명시하지 않으면 이 시스템을 기본 적용. 구현 패턴 상세는 `references/build-patterns.md` 참조.

## 6. 산출물

`부동산투자성향테스트.html` (프로젝트 루트). 브라우저에서 더블클릭으로 열리고 즉시 동작해야 한다. webapp-testing 또는 브라우저로 한 번 열어 두 트랙을 끝까지 풀어보고 결과가 뜨는지 self-check한다.
