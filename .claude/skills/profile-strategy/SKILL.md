# 유형 프로필 & 활용 가이드 작성

두 가지를 만든다: (1) 각 유형이 "내 얘기"로 읽히는 **프로필 카드**, (2) 그 유형이 실제 매매·경매에서 무엇을 골라야 하는지 **활용 가이드**. 마케팅이 아니라 **본인 투자 판단 도구**다.

## 작성 톤

- **균형**: 모든 유형에 강점과 약점이 공존한다. 칭찬 일색 금지.
- **맹점 필수**: 각 유형이 스스로 못 보는 위험(blind spot)을 반드시 한 문단 짚는다. 이게 본 도구의 핵심 가치다.
- **행동 가능**: "당신은 ~경향이 있다"로 끝내지 말고 "그러니 매수/입찰 전 ~를 확인하라"로 연결한다.
- **2인칭 직설**: 투자자 본인에게 말하듯. 군더더기·미사여구 최소.

## 프로필 카드 구조 (`03_profiles.md`, 유형마다)

```
## [유형코드] 유형명 (키워드 3개)
**한 줄 정체성:** ...
**강점:** 2~3개 (구체적 투자행동으로)
**약점:** 2~3개
**맹점 ⚠️:** 스스로 못 보는 위험 한 문단
**잘 맞는 시장 국면:** 상승장/하락장/횡보장 중 어디서 강한가
**보완 파트너 유형:** 약점을 메워줄 반대 성향 유형 코드
```

핵심★ 유형은 깊게, 나머지는 위 골격만 간결히. 컨텍스트를 가치에 비례 분배한다.

## 활용 가이드 구조 (`03_strategy_guide.md`, 유형마다)

매매 유형과 경매 유형을 분리해 서술한다.

**매매 유형 가이드:**
- 적합한 물건 유형 (입지/평형/연식/가격대)
- 피해야 할 함정 (이 성향이 흔히 빠지는 실수)
- 매수 전 체크리스트 (3~5개 구체 항목)
- 보유/매도 전략 힌트

**경매 유형 가이드 (5개 핵심 변수 반영):**
- 적합 물건 + **권리분석 난이도** 권고 (안전입찰 위주냐 특수물건 도전 가능이냐)
- **명도 리스크** 수용 권고
- 입찰 전 체크리스트 (권리분석·시세조사·명도계획·자금계획)
- **개인화 경고**: 주택보유상황(다주택 취득세 중과·양도세, 무주택 실거주 의무·대출한도)과 경매경험(초보는 특수물건·고가낙찰 경고)에 따른 맞춤 주의

## 규제 요약 가이드 섹션

활용 가이드에 **현행 규제 차원 체크리스트**를 포함한다. 근거는 오직 `_workspace/00_regulations.md`(출처·신뢰도 표기 grounding 문서)다. 기억으로 규제 수치를 쓰지 마라.

- **공통 규제 요약 섹션** 1개 + **유형별 규제 주의** 연결: 보유주택수(무주택/1주택/다주택)와 규제지역 여부에 따라 LTV·DSR·대출한도, 취득세·양도세 중과, 전입·실거주 의무, 갭투자·자금조달계획서를 체크리스트로 정리한다.
- **경매 트랙 핵심**: "대출규제는 경매도 동일, 단 토허 실거주의무·갭투자차단은 경매 면제" — 규제회피 기회이자 함정(고가낙찰 쏠림)임을 양면으로 서술한다(00_regulations.md D절).
- **신뢰도·면책**: grounding 문서가 `[불확실]`로 둔 항목(양도세 유예 종료, 6·30 신규지정 범위 등)은 "시점·원문 확인 필요"로 쓰고, "실거래·대출·세금 적용 전 전문가 확인" 면책을 가이드 말미에 둔다. 이 도구는 규제 안내일 뿐 세무·법률 자문이 아니다.

## 웹 결과화면 연동

각 유형의 (유형명 1줄 + 키워드 3개 + summary 1줄)는 diagnostic-designer의 JSON `types[]` 객체에 들어간다. 프로필 작성 시 이 요약을 함께 확정해 SendMessage로 공유한다. 상세 본문은 web-builder가 `03_*.md`에서 직접 가져가므로, 마크다운 헤더(`## [유형코드]`)를 기계가 파싱하기 쉽게 일관되게 쓴다.

## 정합성

유형 코드는 `01_typology.md`와 JSON의 정본을 그대로 쓴다. 새 코드를 만들거나 철자를 바꾸지 않는다. 현실에서 모순되는 유형 조합을 발견하면 typology-architect에게 피드백한다.
