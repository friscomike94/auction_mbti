# 부동산 투자성향 유형 설계

MBTI처럼 **여러 대립축의 조합**으로 투자자 유형을 정의한다. 매매와 경매는 투자 메커니즘이 다르므로 **별도 트랙**으로 설계한다.

## 핵심 원리: 좋은 축의 조건

축(axis)은 두 극(pole)을 가진 대립 스펙트럼이다. 좋은 축은:

1. **양극이 모두 합리적이다** — "안정 vs 공격"은 둘 다 정당한 전략이다. "신중 vs 무모"는 한쪽이 나쁨이므로 축이 아니다. trade-off가 없으면 버린다.
2. **측정 가능하다** — 3~5개의 구체적 상황 문항으로 우세 극을 가릴 수 있어야 한다.
3. **독립적이다** — 다른 축과 강하게 상관되면 중복이다(예: "장기보유"와 "임대수익"이 항상 같이 가면 한 축으로 합친다).
4. **의사결정을 바꾼다** — 그 축의 극이 다르면 실제로 다른 물건/전략을 선택해야 한다.

## 트랙 분리 원칙

| | 매매 트랙 | 경매 트랙 |
|---|---|---|
| 강조 변수 | 입지, 시세차익, 보유기간, 실거주 겸용 | **투자자금·투자용도·투자지역·주택보유상황·경매경험** |
| 리스크 성격 | 시장가격 변동 | 권리분석·명도·유찰/낙찰가 |
| 유형 코드 접두 | `M-` (Maemae) | `A-` (Auction) |

경매 트랙은 사용자가 지정한 5개 핵심 항목(투자자금·투자용도·투자지역·주택보유상황·경매경험)을 **반드시 축 또는 개인화 변수로** 녹여야 한다. 일부는 대립축으로(예: 자금규모 소액↔대액), 일부는 결과 개인화 입력으로(예: 경매경험 횟수) 쓴다.

## 설계 절차

1. **후보 축 검토** — `references/candidate-axes.md`의 매매/경매 후보 축을 읽고 출발점으로 삼는다. 그대로 쓰지 말고 "좋은 축의 조건"으로 검증해 교체·수정한다.
2. **트랙별 4축 확정** — 각 트랙 4개 축을 고른다(4축=16유형). 축이 부족하면 3축(8유형)도 허용. 각 축에 정의·두 극의 의미·trade-off·측정 힌트(어떤 상황 문항으로 잴지)를 적는다.
3. **코드 네이밍** — 각 극에 1글자 코드(겹치지 않게). 유형 코드는 축 순서대로 조합(예: `M-SHLI`). 축 순서를 고정하고 명시한다.
4. **유형 매트릭스** — 가능한 모든 조합을 표로. 각 유형에 1줄 캐릭터 요약. 현실에서 흔하고 의미 있는 **핵심 유형에 ★** 표시(콘텐츠 우선순위).
5. **모순 점검** — 조합 중 현실에서 거의 불가능한 유형(예: 극단 소액인데 대형 특수물건 선호)은 "희귀/모순" 표시하고, 매트릭스 주석에 설명한다.

## 산출물 형식 (`_workspace/01_typology.md`)

```
# 부동산 투자성향 유형 체계

## 매매 트랙
### 축
- [축1 코드] 이름: 극A(코드) ↔ 극B(코드) — 정의 / trade-off / 측정 힌트
... (4축)
### 유형 매트릭스 (축 순서: 축1-축2-축3-축4)
| 코드 | 유형명(가칭) | 1줄 요약 | 핵심★ |

## 경매 트랙
(동일 구조, 5개 핵심 변수 반영 명시)

## 코드 규칙
축 순서, 극 코드표, 접두 규칙
```

축 코드·유형 코드는 **다운스트림(문항·프로필·웹)의 고정 계약**이다. 확정 후 바꾸면 전 산출물에 영향을 주므로, 변경 시 반드시 리더에게 먼저 알린다.
